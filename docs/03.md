### 1. 연말 랠리(Santa Claus Rally) 분석

**쉽게 이해하기**
- **산타 랠리(Santa Claus Rally)** 는 12월 말~1월 초에 주가가 상승하는 경향을 말합니다.
- 12월 평균 수익률이 연간 평균보다 통계적으로 유의미하게 높은지 검증해야 진짜 효과인지 알 수 있습니다.

**미니 예시**
```python
# 12월 평균 수익률이 연간 평균보다 높은지 확인 (예시 데이터)
monthly_avg = {
    1: 0.8,  2: 0.1,  3: 0.5,  4: 1.2,
    5: -0.3, 6: 0.2,  7: 0.9,  8: -0.5,
    9: -1.1, 10: 0.4, 11: 1.5, 12: 1.8
}  # 월별 평균 일별 수익률(%) 예시
year_avg = sum(monthly_avg.values()) / 12
dec_avg  = monthly_avg[12]
print(f'12월 평균 수익률: {dec_avg:.1f}%')
print(f'연간 월평균:      {year_avg:.2f}%')
print(f'산타 랠리:        {"확인됨" if dec_avg > year_avg else "불명확"}')
```

### 2. 월별 효과(January Effect 등) 분석

**쉽게 이해하기**
- **1월 효과(January Effect)** 는 1월에 소형주·전체 시장이 강세를 보이는 경향으로, 연말 세금 매도 이후 재매수 때문으로 설명됩니다.
- 월별 평균 수익률 히트맵을 그리면 어느 달이 강세/약세인지 한눈에 파악할 수 있습니다.

**미니 예시**
```python
# 월별 평균 수익률 비교 (January Effect 확인)
monthly_ret = {
    '1월': 1.2,  '2월': 0.1,  '3월': 0.8,  '4월': 1.5,
    '5월': -0.2, '6월': 0.3,  '7월': 1.0,  '8월': -0.4,
    '9월': -1.0, '10월': 0.6, '11월': 1.8, '12월': 1.6
}  # S&P500 장기 평균(예시, %)
best  = max(monthly_ret, key=monthly_ret.get)
worst = min(monthly_ret, key=monthly_ret.get)
print(f'1월 평균 수익률: {monthly_ret["1월"]:+.1f}%')
print(f'가장 강한 달:    {best} ({monthly_ret[best]:+.1f}%)')
print(f'가장 약한 달:    {worst} ({monthly_ret[worst]:+.1f}%)')
```

### 3. 요일 효과(Monday Effect 등) 분석

**쉽게 이해하기**
- **월요일 효과(Monday Effect)** 는 월요일 수익률이 다른 요일보다 낮은 경향으로, 주말 뉴스 누적이 원인으로 지목됩니다.
- 요일별 평균 수익률을 비교하면 어떤 요일에 매수/매도하는 것이 유리한지 파악할 수 있습니다.

**미니 예시**
```python
# 요일별 평균 수익률 비교 (Monday Effect 확인)
weekday_ret = {
    '월요일': -0.05,
    '화요일':  0.08,
    '수요일':  0.06,
    '목요일':  0.04,
    '금요일':  0.07,
}  # 예시 데이터 (단위: %)
for day, ret in weekday_ret.items():
    bar = '▲' if ret >= 0 else '▽'
    print(f'{day}: {bar} {ret:+.2f}%')

worst_day = min(weekday_ret, key=weekday_ret.get)
print(f'\n→ 가장 약한 요일: {worst_day} → 월요일 효과 {"확인됨" if worst_day == "월요일" else "불명확"}')

# 실전: df["weekday"] = df.index.weekday  (0=월, 4=금)

```

### 4. 실습: 코스피/S&P500 계절성 패턴 Python 분석

**쉽게 이해하기**
- yfinance로 두 지수를 가져와 월별·요일별 평균 수익률을 비교하면 한국과 미국 시장의 계절성 차이를 확인할 수 있습니다.
- `groupby("month")["ret"].mean()` 한 줄로 월별 효과를 바로 추출할 수 있습니다.

**미니 예시**
```python
# yfinance로 코스피/S&P500 계절성 비교 (실전 코드)
import yfinance as yf

for ticker, name in [("^GSPC", "S&P500"), ("^KS11", "KOSPI")]:
    df = yf.download(ticker, start="2010-01-01", progress=False)
    df["month"]   = df.index.month
    df["weekday"] = df.index.weekday   # 0=월, 4=금
    df["ret"]     = df["Close"].pct_change()

    monthly = df.groupby("month")["ret"].mean() * 100
    best_m  = monthly.idxmax()
    worst_m = monthly.idxmin()

    months = ['1월','2월','3월','4월','5월','6월',
              '7월','8월','9월','10월','11월','12월']
    print(f"\n{name}")
    print(f"  강세 달: {months[best_m-1]} ({monthly[best_m]:+.3f}%)")
    print(f"  약세 달: {months[worst_m-1]} ({monthly[worst_m]:+.3f}%)")
    print(f"  9월 평균: {monthly[9]:+.3f}%  ← Sell in September 검증")
```

### 💾 데이터를 저장해두는 냉동고: `app/services/data_cache.py`

계절성 분석은 **여러 해의 데이터**가 필요해요. 매번 인터넷에서 새로 받으면 느리니까,  
웹앱은 데이터를 MongoDB에 **냉동 보관**해뒀다가 인터넷이 없을 때도 꺼내 쓸 수 있어요.

```python
# 📁 app/services/data_cache.py

async def cache_get(key: str, max_age_hours: float = 24):
    # MongoDB에서 데이터를 꺼내요
    doc = await db["data_cache"].find_one({"key": key})
    if not doc:
        return None          # 저장된 데이터 없음 → 인터넷에서 새로 받아야 해요

    age = 지금시간 - doc["updated_at"]
    if age > max_age_hours:
        return None          # 너무 오래된 데이터 → 새로고침 필요
    return doc["data"]       # 신선한 데이터 반환!

async def cache_set(key: str, data):
    # MongoDB에 데이터를 저장해요 (= 냉동 보관)
    await db["data_cache"].update_one(
        {"key": key},
        {"$set": {"data": data, "updated_at": 지금시간}},
        upsert=True
    )
```

### 🔄 1시간마다 자동 수집: `app/services/sync_scheduler.py`

웹앱은 인터넷이 연결된 순간, 자동으로 KOSPI와 S&P 500을 받아 MongoDB에 저장해요.  
이 데이터를 모아두면 나중에 계절성 분석 재료로 쓸 수 있어요!

```python
# 📁 app/services/sync_scheduler.py

# 계절성 분석에 필요한 거시경제 지표들을 자동으로 수집
async def _sync_macro_indicators():
    # KOSPI, S&P500, VIX, 금리, 환율 등 9개 지표를 Yahoo Finance에서 가져옴
    tasks = [_yahoo_latest(m["symbol"], m["unit"]) for m in MACRO_SYMBOLS]
    results = await asyncio.gather(*tasks)
    await cache_set("macro_indicators", results)   # MongoDB에 저장!
    # → 이 데이터로 "이번 달이 평균보다 강세인가?" 분석 가능
```

> **초등생 비유:** 마치 **자동 날씨 기록기**가 매 시간마다 온도를 측정해서 일기장에 적어두는 것과 같아요.  
> 나중에 "지난 10년 동안 12월은 평균적으로 따뜻했나?" 를 알기 위해서예요!

### 📊 웹앱 "거시경제" 메뉴에서 계절성 패턴 확인하기

**로보 어드바이저 → 거시경제 지표** 메뉴를 누르면 KOSPI와 S&P500의 현재 수익률을 볼 수 있어요.  
이 데이터를 MongoDB에 쌓다 보면 자연스럽게 계절성 패턴 분석 재료가 만들어져요:

```
MongoDB data_cache 컬렉션
├── key: "macro_indicators"          ← 매 1시간마다 업데이트
│   data: [
│     { symbol: "^GSPC", price: 5280, change_p: +0.5 },   ← 오늘 S&P500
│     { symbol: "^KS11", price: 2650, change_p: -0.2 },   ← 오늘 KOSPI
│     ...
│   ]
│   updated_at: "2026-05-17 09:00"
├── key: "market_indices"            ← 시장 지수 (화면 상단 티커)
└── key: "us_stocks"                 ← 미국 주요 종목 10개
```

> **초등생 비유:** MongoDB는 **주식 일기장**이에요.  
> 매 시간 날씨(주가)를 적다 보면, 나중에 "봄에는 항상 비가 오네!" 처럼  
> "11월에는 주가가 항상 오르네!" 를 발견할 수 있어요 — 그게 바로 계절성 패턴이에요!
